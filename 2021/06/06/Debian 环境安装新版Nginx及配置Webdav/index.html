4. make upgrade 平滑升级

原因是mount.davfs命令在挂载时和 WebDAV 服务器建立连接，并通过 libneon库的 ne_options 函数发送 HTTP OPTIONS 请求获取 WebDAV 服务器能力，虽然返回成功但判断坚果云 WebDAV 服务器不支持 Class 1 WebDAV，因此直接报错挂载失败。
看到这里解决办法也就简单了，davfs2 提供了通过配置文件禁用 WebDAV头检测的功能，
直接修改/etc/davfs2/davfs2.conf配置文件解注释并改为：ignore_dav_header 1
